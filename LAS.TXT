

     Описание формата LAS - Log ASCII Standart (версия 1.20)

     1. Общее описание.
     # Гибкие диски в формате LAS должны быть доступны для MS/DOS
или PC/DOS.
     # Файл должен быть записан в коде ASCII. Если файл записан в
бинарном или другом виде,  на  гибком  диске  должен  размещаться
EXE-файл, который конвертирует обратно в формат LAS.
     # Файл должен помещаться на одной дискете. Если он не разме-
щается на  одной дискете,  его необходимо разбить на нужное число
файлов. Это может быть достигнуто, например, сегментированием ин-
тервалов глубин.
     # Каждый файл должен иметь расширение ".LAS".
     # LAS - минимальный стандарт и допускает расширения по жела-
нию пользователей.

     2. Основные компоненты файла каротажных кривых.
     Каждый файл содержит до 6 разделов. Каждый раздел начинается
энаком '~'. Эти разделы не имеют определенного порядка следования
(некоторые иэ  них могут быть опущены) за исключением последнего.
который всегда должен быть разделом  данных.  Возможны  следующие
разделы: ~V, ~W, ~C, ~P, ~O, ~A.
     3.Флаги используются для  того,  чтобы  помочь  пользователю
идентифицировать строки в файле.
а)'~' (ASCII код - 126).  При его использовании  он  должен  быть
первым не-пробелом и не-кавычкой на строке.  Используется для то-
го, чтобы пометить начало раздела в файле.  Первый  символ  сразу
после тильды  идентифицирует раздел.  Все заглавные буквы на этом
месте зарезервированы для использования разработчиками стандарта.
Остальная часть строки воспринимается как комментарий.
б)'#' (ASSCII код - 35). При его использовании  он  должен  быть
первым не-пробелом и не-кавычкой на строке. Используется для ука-
зания, что строка является комментарием.  Строки комментариев мо-
гут появляться всюду, кроме раздела данных.
в)'.' и ':'.  В разделах. кроме раздела данных, точки и двоеточия
используются для  разделения информации на строке (с целью облег-
чить чтение строки).  Информация слева от двоеточия - это дуталь-
ное определение  мнемоники,  которая записана слева от двоеточия.
Точка используется для разделения двух мнемоник.  справа и  слева
от точки и двоеточия могут встречаться пробелы.
     4.Описания разделов.
     4.1. ~Vercion  information  -  информация  о версии системы.
Раздел должен содержать следующие две строки.
     VERS.            1.20:CWLS log ASCII Standart - VERCION 1.20
   и WRAP.             YES: Multiple lines per depth step
или
     WRAP               NO: One line per depth step
     Если no wrap используется,  то строка может иметь максималь-
ную длину 256 символов (включая возврат каретки и переход на сле-
дующую строку).  Если yes wrap используется,  то значение глубины
должно располагаться на отдельной строке,  а  все  строки  данных
должны быть не длиннее 80 символов (включая возврат каретки и пе-
реход на следующую строку).
     4.2. ~Well information  -  информация  по  скважине.  Раздел
идентифицирует скважину,  ее расположение и указывает начальную и
конечную глубину интервала данных в файле. Этот раздел должен со-
держать следующие строки с мнемоникой, как она записана ниже.
          STRT.M        nnn.nn:
Начальная глубина  интервала  данных в файле.  "nnn.nn" указывает
значение глубины   Число  десятичных  знаков не ограничено.  ".М"
указывает, что значение глубины приведено в метрах.  Можно  заме-
нить мнемонику при использовании других единиц измерения.
          STOP.M        nnn.nn:
Конечная глубина  интервала  данных  в файле.  "nnn.nn" указывает
значение глубины   Число  десятичных  знаков не ограничено.  ".М"
указывает, что значение глубины приведено в метрах.  Можно  заме-
нить мнемонику при использовании других единиц измерения.  Конеч-
ная глубина может быть больше или меньше начальной.
          STEP.M        nnn.nn:
Шаг квантования  данных  по глубине,  "nnn.nn" указывает значение
шага. Знак "-" должен предшествовать шагу, если начальная глубина
больше, чем конечная глубина.  Нулевое значение шага указывает на
переменный шаг данных по глубине.
          NULL.         -nnn.nn:
Код отсутствия информации. Обычно равен -9999 или -999.25.
        COMP.           COMPANY:Название организации
        WELL.              WELL:Наименование скважины
        FLD .             FIELD:Наименование площади
        LOC .          LOCATION:Расположение скважины
        PROV.          PROVINCE:Название провинции
Для областей за пределами Канады эта строка может быть заменена на
            CNTY.              COUNTY:
            STAT.               STATE:
            CTRY.             COUNTRY:
        SEVC. SERVICE COMPANY:Наименование организации, проводив-
                                                шей каротаж
        DATE.          LOG DATE:Дата регистрации каротажных данных
        UWI .       UNIQUE WELL:Уникальный идентификатор скважины
Для областей за пределами Канады это может быть заменено на
        API .        API NUMBER:
Допустимы дополнительные строки. Ограничение на число строк нет.
     4.3.~Curve information.  Этот раздел описывает кривые и еди-
ницы измерений в том порядке,  в котором они размещены в  разделе
данных этого файла.  Используемая мнемоника не имеет ограничений,
но должна быть определена на той строке, на которой она появляет-
ся. API коды не обязательны.
Кривые, описанные в этом разделе должны быть  обязательно  предс-
тавлены в разделе данных. Первая описанная кривая обычно глубина.
Формат:
     aaaa.bbbb                           : curve description
если не используются API коды, и
     aaaa.bbbb              nn mmm oo pp : curve description
где
     aaaa - мнемоника кривой
     bbbb - единицы измерений
     nn   - API log type code
     mmm  - API curve type code
     oo   - API curve class code
     pp   - API curve modifier code
если используются API коды.
     4.4.~Parametr information block.  Этот раздел не обязателен.
Он описывает параметры или константы и их значения.  Используемая
мнемоника не имеет ограничений,  но должна быть определена на той
строке, на  которой она появляется.  Ограничений на число строк в
разделе нет. Формат:
     aaaa.bbbb             nnn.nn
где
     aaaa - мнемоника кривой
     bbbb - единицы измерений
     nnn.nn - значение параметра
Пример:
     DFD.K/M3                1200: Mud Weight
     4.5.~Other information. Этот раздел не обязателен. Он введен
как область комментариев.
     4.6.~ASCII Log Data.  Этот раздел всегда должен быть послед-
ним разделом файла. Глубины всегда должны появляться в первой ко-
лонке строки. Каждая колонка данных должна быть отделена по мень-
шей мере одним пробелом.
Wrap мода должна использоваться, если данные не помещаются на од-
ной строке  длиной  256 символов (с учетом символа конца строки и
возврата каретки).
Если warp мода используется, то:
     - десятичные точки должны использоваться для облегчения чте-
ния;
     - глубина должна располагаться на отдельной строке;
     - длина каждой строки данных не должна превышать 80 символов
(с учетом символа конца строки и возврата каретки).
Не допускается использование экспонент (формата Е). Раздел ~С мо-
жет быть использован для устранения этого ограничения путем изме-
нения единиц измерений.
